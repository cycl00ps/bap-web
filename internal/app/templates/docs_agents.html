Agent Instructions

Use these resources when connecting an LLM or automation agent to BAP Web.

Start Here

Agents should use a user-supplied API token. Do not ask the platform for unauthenticated access or try to create your own token unless the user explicitly asks.

Markdown guide OpenAPI JSON Swagger UI llms.txt

Authentication

Use the bearer token supplied by the user or administrator:

Authorization: Bearer bap_...

Prefer short-lived non-admin tokens for VM work. Token expiry defaults to 30 days. Use admin tokens only for image, kernel, token, and other administrative operations.

Recommended Agent Workflow

  1. Read /docs/agents.md and /openapi.json.
  2. Call GET /api/session with the supplied bearer token to confirm identity.
  3. Discover SSH keys, base images, kernels, networks, and egress policies.
  4. Create and start VMs only when the user explicitly asks for that work.
  5. Use POST /api/vms/{id}/exec for shell commands and inspect exit codes.
  6. Clean up created VMs unless the user asks to keep them running.
